## Key Work

### Crescent — SillyTavern Compatibility Architecture

The major session (7f1caf92, 16 hours output) dived deep into designing a SillyTavern-adjacent system with focus on:

- **Lorebook system**: How to implement ST-compatible lorebook JSON structure, including entry fields, priority ordering, keyword matching, and trimming strategies
- **Context assembly**: Token budget management, efficient keyword matching (using NFAs), preservation priority (history vs lorebook entries), and comparison with ST's approach
- **Macro system**: {{char}} and {{user}} substitution, case insensitivity ({{Char}}, {{CHAR}}), legacy angle-bracket syntax (optional), and whether to embed full macro library or keep it lightweight
- **Card structure**: Field layout, information density in editing UX, and ST feature parity requirements
- **Mobile UX**: Swipe gestures for branch navigation, long-press constraints, message editing/deletion affordances on touch devices
- **Input handling**: "Impersonate" and "continue" buttons (configurable, default hidden), swipe-based actions, message prefilling for continuation

Extended conversation thread focused on questioning design choices, pushing back on self-confident suggestions from the AI model, and insisting on thinking through user workflows (worldbuilding use cases, reader vs author perspectives, what actually matters for usability).
