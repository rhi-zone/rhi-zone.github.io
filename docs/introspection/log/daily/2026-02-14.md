## Two major threads

### 1. GML Decompiler Correctness — Bounty (Game Maker project)

Early morning through afternoon: systematic bug hunting and fixing in the reincarnate GML frontend. Pattern: load reference gamemaker output (`~/git/bounty/`), compare against decompiled output (`~/Bounty/out/`), identify runtime errors, fix the IR/lowering logic.

**Bugs addressed:**

- **Global variable access fix** — GML scripts were emitting `global.field` instead of `self.field`. Fixed by changing the semantic context for script execution.
- **Global variable type inference** — Post-fix, all global accesses needed type inference since they came through at runtime. Added inference logic.
- **`variable_global_get/set` optimization** — Replaced function call pattern with direct property access (`global.x`).
- **GML `argument[N]` parameter access bug** — Functions accessing `argument0`, `argument1` etc. produced completely wrong comparisons. Root cause: parameter binding wasn't connecting to the argument array. Major fix.
- **Loop counter initialization** — `repeat(N)` loops generated infinite loops because the counter variable was being replaced by its initial value during constant folding. Needed escape analysis in `promote_single_store()`.
- **GML const folding regression** — `fold_single_use_consts` was incorrectly folding loop-carried variables.
- **Sprite constant resolution** — `sprite_index` field defaults should resolve to named constants in output.

**Operational note:** Multiple sessions ran GML backlog items in parallel. User kept asking "what's next?" and picking items from the TODO list. Session workflow: spawn new session, "Implement the following plan: [plan from TODO.md]", execute, commit, loop.

**Test coverage gap discovered:** Several bugs weren't caught by existing tests. Sessions were asking "is this a regression?" after finding issues. Discussion about adversarial tests and whether test coverage is sufficient.
