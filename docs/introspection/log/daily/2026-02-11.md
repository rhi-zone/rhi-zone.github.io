### reincarnate (Flash→TypeScript codegen) — ~5 sessions, spread across day (14:26–18:31 UTC)

Heavy debugging of runtime errors after initial implementation of AVM2 features:

- **Circular import issues**: Multiple sessions (2891fc7c, 561c80e7, 4dc659fa) tackling esbuild module flattening breaking class inheritance when Monster.ts statically imports types from files that import Monster. Attempted fixes: topological sort of barrel exports, late-bound type checks, "generalize circular import detection" — persisted across multiple turns despite multiple approaches.
- **Type checking and casting semantics**: Separated `AsType` (AS3 type cast) from `Coerce`/`Convert` (JS coercion). Debated whether AS3 interfaces should be exported as abstract classes and resolved at runtime via `isType()` helper. User pushed back on "aren't we over-specializing the IR for AS3?" — settled on interface-as-abstract-class approach.
- **Global scope pollution**: Fixed ESM incompatibility where `export const mx_internal: any;` without initializer is invalid — switched to `let`/`var`. Similarly fixed `kGAMECLASS` assignment-to-import error.
- **Runtime gaps**: Missing AS3 globals like `int`, `uint` needed at runtime; rest parameter support incomplete. Multiple "ReferenceError: X is not defined" crashes (GENDER_NONE, int, Utils, etc.).
- **Scope/initialization bugs**: Block→Loop transitions dropping variable initializers; scope stack cleared at block boundaries causing `null.slotN` access; struct literal emission of `ClassDef` sites needed updates across scaffold.rs, core transforms, and tests.
- **Test infrastructure**: Multiple agent tasks spawned for bulk ClassDef updates (17+ sites in emit.rs tests alone), with task completion notifications appearing mid-session.

Pattern: systematic debugging from "ReferenceError" chain. User patient but frustrated ("didn't help :/", "what the hell ●"). Heavy reliance on agent-based bulk edits to keep up with volume of references needing updates. The session reads like grinding through a long tail of integration issues after a major refactor.
