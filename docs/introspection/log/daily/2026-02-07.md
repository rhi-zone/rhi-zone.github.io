### Reincarnate Work Patterns

The day was characterized by **problem-spotting → patch → move-to-proper-place** cycles:

1. **Scope lookup resolution** — eliminated `Flash_Scope.findPropStrict` calls systematically
   - Resolved class member lookups to `this`
   - Stripped scope lookups from `GetField`/`SetField`
   - Addressed unqualified globals (rand, game, outputText, etc.)

2. **Redundant assignment elimination** — thousands of `vN = vM;` assignments from SSA block arguments
   - Worked on merging block parameter assignments properly
   - Trailing merge assign position fixes

3. **Type inference** — became clear this was a blocker for further progress
   - `as boolean` casts on method calls despite return type info available
   - ~445 `:any` annotations in Flash output
   - Multi-typed locals (e.g., `race = 0.0` then `race = "human"`)
   - Decision: need constraint-based solver, not just forward dataflow

4. **Control flow patterns** — caught problematic emit patterns:
   - Double negation in loops: `if (!(!v55)) {`
   - Empty branches
   - Redundant conditional phis
   - Short-circuit operators (`||`/`&&`) lowered prematurely

5. **Local variable names** — AVM2 bytecode does provide this info; currently discarded by Flash frontend

### Architectural Concerns Surfaced

- Ternary detection (`try_ternary_assigns`)
